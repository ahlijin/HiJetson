# Changelog / 更新日志

## 0.9.3 (2026-06-04)

### 新增
- **唤醒提示音节点 (voice_feedback)**
  - 新包: package.xml, setup.py, setup.cfg, voice_feedback_node.py
  - 订阅 `/voice/wake_word` — 检测到唤醒词时播放上升双音 (800→1200Hz)
  - 订阅 `/voice/asr_result` — 识别完成时播放确认短音 (1000Hz)
  - 通过 `voice_params.yaml` 配置音量和输出设备

### 改进

**音频架构全线改为 PulseAudio**
- `voice_capture.device_index`: `0` (ALSA直连) → `-1` (PulseAudio默认)
- `voice_feedback.device_index`: `1` (HDMI 1 ALSA) → `-1` (PulseAudio默认)
- 砍掉 `pasuspender` 和 `pactl suspend-source`，启动不再需要手动释放设备
- 麦克风和蓝牙输出全部走 PulseAudio 同一通道，无冲突

**VAD 门控唤醒词 (省 CPU)**
- `voice_wake_word` 新增 `/voice/voice_activity` 订阅
- 只在 VAD 检测到人声时才跑 OpenWakeWord 推理
- 静音时直接 return，省 ~90% CPU 推理开销
- `_vad_active = True` 作为默认值，兼容尚未启动 VAD 的场景

**Whisper 幻觉过滤**
- ASR 节点新增原始 RMS 能量检查 (`threshold: 0.005`)
- 在归一化之前判断：能量太低 → 噪音片段 → 直接跳过
- 避免 Whisper 在接近静音的音频上产生幻觉识别

### 修复
- 蓝牙连接后唤醒提示音不响
  - 原因: feedback 写死 HDMI 1 ALSA 设备，跳过 PulseAudio
  - 修复: 改为 PulseAudio 默认输出 (`device_index: -1`)
- `pasuspender` 杀死 PulseAudio 导致蓝牙失效
- 偶尔出现 Whisper 幻觉识别问题

### 配置
- `voice_params.yaml` 新增 `voice_feedback` 参数段
  - `device_index`: 输出设备 (默认 -1 = PulseAudio 默认)
  - `wake_volume`: 唤醒提示音量 (0.5)
  - `result_volume`: 识别提示音量 (0.3)
  - `sample_rate`: 输出采样率 (48000)
- `voice_params.yaml` 变更
  - `voice_capture.device_index`: `0` → `-1`
  - `voice_feedback.device_index`: `1` → `-1`

### 文件变更

- Capture 通过 PulseAudio 默认设备打开成功
- 反馈音通过 PulseAudio 默认输出 (蓝牙) 播放正常
